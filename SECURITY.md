# Security Policy for Spareflow AI

We take the security of Spareflow AI seriously. This document outlines how to report vulnerabilities and which versions are actively supported.

---

## Supported Versions

Only the latest active major/minor release is supported with security patches:

| Version | Supported |
| :--- | :--- |
| **`v1.x.x`** |  Yes |
| `< v1.0.0` |  No |

---

## Reporting a Vulnerability

If you find a security bug, please **do NOT** open a public GitHub Issue. Instead, report it privately to the maintainers:

1. Send an email to the security email contact listed in our repository settings or contact the repository owner directly.
2. Provide a clear description of the issue, steps to reproduce, or a proof-of-concept (PoC).
3. We will acknowledge receipt of your report within 48 hours and coordinate a patch or mitigation strategy before public disclosure.
